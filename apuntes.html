Permisos y Roles más Usados:

  • givePermissionTo(): nos permite dar seguimiento a un usuario o rol
  • revokePermissionTo(): Revocar el permiso de un usuario o rol
  • hasPermissionTo(): comprueba si un usuario o rol tiene un permiso determinado
  • assignRole(): Asigna rol a un usuario
  • removeRole(): elimina el rol de un usuario
  • hasRole(): comprueba si un usuario tiene un rol
  • hasAnyRole (Role :: all ()): comprueba si un usuario tiene alguna de una lista de roles dada
  • hasAllRoles (Role :: all ()): comprueba si un usuario tiene toda una lista de roles determinada

Creating Migrations

To create a migration, you may use the migrate:make command on the Artisan CLI:

php artisan migrate:make create_users_table

The migration will be placed in your app/database/migrations folder, and will contain a timestamp which allows the framework to determine the order of the migrations.

You may also specify a --path option when creating the migration. The path should be relative to the root directory of your installation:

php artisan migrate:make foo --path=app/migrations

The --table and --create options may also be used to indicate the name of the table, and whether the migration will be creating a new table:

php artisan migrate:make add_votes_to_user_table --table=users

php artisan migrate:make create_users_table --create=users

Running Migrations

Running All Outstanding Migrations

php artisan migrate

Running All Outstanding Migrations For A Path

php artisan migrate --path=app/foo/migrations

Running All Outstanding Migrations For A Package

php artisan migrate --package=vendor/package
